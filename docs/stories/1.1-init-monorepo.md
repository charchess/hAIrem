# Story 1.1: Initialiser le repo Monorepo

**Epic:** 1. Foundation & H-Core Architecture
**Status:** Ready for Dev
**Goal:** Créer la structure de base du projet pour permettre le développement parallèle du Backend (H-Core) et du Frontend (A2UI).

## Acceptance Criteria
- [ ] La structure de répertoires suivante existe :
    ```
    hAIrem/
    ├── apps/
    │   ├── h-core/        # Backend Python (FastAPI/Async)
    │   └── a2ui/          # Frontend (Web)
    ├── agents/            # Dossiers des experts
    ├── packages/          # Code partagé (si besoin)
    └── docker-compose.yml
    ```
- [ ] Le fichier `docker-compose.yml` est créé et lance un service `h-core` (Python) et un service `redis`.
- [ ] Le projet est initialisé avec `poetry` ou `pip` (fichier `pyproject.toml` ou `requirements.txt`).
- [ ] Un linter (Ruff ou Flake8) est configuré.

## Implementation Tasks
1.  **Scaffolding :** Créer les dossiers `apps/h-core`, `apps/a2ui`, `agents`.
2.  **Dependency Management :** Initialiser un projet Python dans `apps/h-core` (recommandé: Poetry). Ajouter `fastapi`, `uvicorn`, `redis`.
3.  **Docker :** Créer le `Dockerfile` pour `h-core` et le `docker-compose.yml` racine incluant l'image Redis officielle (alpine).
4.  **Linting :** Ajouter un fichier de configuration pour le linter (ex: `ruff.toml`).
5.  **Validation :** Vérifier que `docker-compose up` lance les conteneurs sans erreur.
