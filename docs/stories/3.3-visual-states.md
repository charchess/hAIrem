# Story 3.3: Gestion des états visuels (Interactivity)

**Status:** Draft
**Story:**
**As a** User,
**I want** the agent to change its pose when I speak or when it processes information,
**so that** the interaction feels reactive and alive.

## Acceptance Criteria
1.  Définition des états visuels : `IDLE`, `LISTENING` (User speaking), `THINKING` (LLM processing), `SPEAKING` (Streaming response).
2.  L'UI change de pose instantanément lors de la détection d'une activité utilisateur (Mockée pour le moment).
3.  Support du streaming de texte : affichage caractère par caractère.

## Tasks / Subtasks
- [ ] Task 1: State Machine UI (AC: 1)
    - [ ] Implémenter une machine à états simple dans le JS Frontend.
- [ ] Task 2: Typewriter Effect (AC: 3)
    - [ ] Créer une fonction utilitaire pour l'affichage progressif du texte.
- [ ] Task 3: Trigger Logic (AC: 2)
    - [ ] Lier les messages H-Link de type `system.log` ou `narrative.text` aux changements d'états.
