# Story 1.2: Configurer le Bus Redis

**Epic:** 1. Foundation & H-Core Architecture
**Status:** Ready for Dev
**Goal:** Établir le canal de communication asynchrone entre les composants du système.

## Acceptance Criteria
- [ ] Le service Redis est accessible depuis le conteneur `h-core`.
- [ ] Une classe `RedisClient` (ou équivalent) est implémentée dans `h-core` pour gérer la connexion.
- [ ] Un script de test démontre la capacité de publier un message sur un channel et de le recevoir (Pub/Sub).

## Implementation Tasks
1.  **Redis Connection :** Dans `apps/h-core/src/infrastructure/redis_client.py` (chemin suggéré), implémenter une classe singleton ou un fournisseur de dépendance pour la connexion Redis. Utiliser `redis-py` en mode async.
2.  **Configuration :** Utiliser des variables d'environnement (chargées depuis `.env` ou Docker) pour l'hôte et le port Redis (`REDIS_HOST`, `REDIS_PORT`).
3.  **Pub/Sub Logic :** Ajouter des méthodes `publish(channel, message)` et `subscribe(channel, callback)` à la classe client.
4.  **Test :** Créer un script `scripts/test_redis.py` qui lance un subscriber en background, publie un message "Hello hAIrem", et vérifie la réception.
