# 1. Record Architecture Decisions

Date: 2026-01-27

## Status

Accepted

## Context

Nous avons besoin d'un processus pour enregistrer les décisions architecturales importantes (choix de technologies, structures de données, protocoles) afin de garder une trace du "pourquoi" et éviter de refaire les mêmes débats.

## Decision

Nous utiliserons les **Architecture Decision Records (ADR)**.
Les fichiers seront stockés dans `docs/architecture/decisions/` et numérotés séquentiellement.

## Consequences

*   Toute décision structurante doit faire l'objet d'un fichier Markdown dans ce dossier.
*   Cela facilite l'onboarding des nouveaux développeurs.
